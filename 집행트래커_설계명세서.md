# 집행 트래커 (모듈 2) — 설계 명세서

**제품명**: 과제온 > 집행 트래커
**작성일**: 2026-04-07
**버전**: v0.1 (MVP 프로토타입)

---

## 1. 목적

정부 R&D 과제를 수행하는 1~5인 스타트업 담당자가 예산 집행 현황을 실시간으로 파악하고, 증빙 누락·참여율 초과 등의 리스크를 사전에 방지할 수 있게 한다.

## 2. 사용자

- 1인~5인 스타트업의 대표 또는 겸직 경영관리 담당자
- 정부 R&D 과제를 처음 수행하거나 경험이 적은 사람
- 엑셀로 예산 관리를 해오던 사람

## 3. 화면 구성

### 3-1. 과제 설정

**입력 방식 2가지:**
- 직접 입력: 기관 선택(중기부/산업부) → 과제명, 수행 기간, 마감일 → 비목별 예산 수동 입력
- 문서 업로드: 공고문 또는 사업수행계획서 업로드 → AI가 기관·비목·금액 자동 추출

**예산 구조:**
- 총 사업비 = 정부출연금 + 민간부담금(현금) + 민간부담금(현물)
- 민간부담금 자동 계산: "우리 회사가 준비해야 할 금액" 표시

**기관별 비목 틀:**
- 중기부: 인건비 / 직접비(재료비, 수탁연구개발비, 연구활동비 등) / 간접비 / 위탁연구개발비
- 산업부: 인건비 / 연구시설장비비 / 연구재료비 / 연구활동비 / 연구수당 / 위탁연구비 / 간접비

### 3-2. 대시보드

- 비목별 잔액 막대그래프 (예산 대비 집행 비율)
- 협약예산 vs 변경예산 비교 표시
- 월별 집행률 추이 차트
- 참여율 현황 (100% 초과 시 빨간 경고)
- 마감일 D-day 카운트다운
- 증빙 누락 건수 요약

### 3-3. 집행 등록

- 입력 항목: 비목 선택, 금액, 일자, 적요(메모)
- 등록 즉시 비목별 잔액 재계산
- 초과 시 즉시 경고 표시
- 해당 비목의 기관별 필요 증빙 서류 체크리스트 자동 표시

### 3-4. 증빙 현황

- 전체 집행건의 증빙 완료/미완료 목록
- 마감일 기준 긴급도 색상 구분:
  - D-30 이상: 초록
  - D-14~D-30: 노랑
  - D-7 이내: 빨강
- 증빙 체크리스트 항목별 체크 가능

## 4. 기관별 증빙 서류 데이터

### 중기부
| 비목 | 필요 증빙 |
|------|-----------|
| 인건비 | 급여대장, 이체확인증, 4대보험 납부확인서 |
| 재료비 | 세금계산서, 통장사본, 내부품의서 |
| 여비 | 출장신청서, 출장보고서, 교통비영수증 |
| 회의비 | 회의록, 영수증, 참석자명단 |
| 위탁연구개발비 | 위탁계약서, 세금계산서, 연구보고서 |

### 산업부
| 비목 | 필요 증빙 |
|------|-----------|
| 인건비 | 급여대장, 이체확인증, 원천징수영수증 |
| 연구시설장비비 | 견적서, 세금계산서, 자산등록대장 |
| 연구재료비 | 세금계산서, 거래명세서, 내부품의서 |
| 연구활동비 | 영수증, 사용내역서 |
| 위탁연구비 | 위탁계약서, 세금계산서, 결과보고서 |

## 5. 프로토타입 범위

- 프론트엔드 단독 동작 (React)
- 데이터는 인메모리 상태 관리 (localStorage 미사용)
- 문서 업로드 AI 추출은 UI만 구현 (실제 AI 연동은 다음 단계)
- 샘플 데이터 포함하여 즉시 체험 가능
