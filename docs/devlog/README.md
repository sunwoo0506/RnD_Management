## ✍️ 작성 원칙 (비개발자 배려)

1. **전문용어는 처음 나올 때 괄호로 풀어쓰기**
   예: `API(프로그램끼리 데이터를 주고받는 통로)`
2. **항상 '무엇을 · 왜 · 결과' 3가지를 채우기**
3. **문제는 증상 → 원인 → 해결 순서로 쓰기**
4. **이모지로 섹션을 나눠 긴 글도 부담 없이 읽히게 하기**
