- **왜:** 어제 팁스 지침의 증빙 표를 규정DB에 이미 넣었고 자동 테스트도 통과하는데, 정작 실제 화면에서는 그대로였습니다. 코드가 맞는데 화면이 틀리면 **데이터가 오는 길**을 봐야 합니다.

```

tips2026-general    조문  16  증빙표 있음   ← 내 컴퓨터
```

### 3. 빠져 있던 원문 표를 규정DB에 싣기
- **무엇을:** 통합관리지침의 `제36조 <표-10> 창업기업등 사업비 비목`(비목별 정의·증빙서류 표)과 `세부관리기준 제21조`를 규정DB에 추가했습니다.
- **왜:** 증빙 21건이 전부 이 표 하나에서 나왔는데, 정작 그 표가 원문 보관소에 없었습니다. 표라는 이유로 "조문이 아니다"라며 빠뜨린 것으로 보입니다.
- **결과:** 두 건 모두 **순수 추가**라 기존 근거 연결은 하나도 건드리지 않았습니다. 예비창업 팩의 원문이 16개 → 18개가 되고, 증빙 근거 39건이 전부 자기 사업 안에서 해결됩니다.

### 4. 어느 문서가 진짜 출처인지 먼저 확인

- **왜:** 같은 내용의 표가 두 문서(통합관리지침 / 세부관리기준)에 모두 있어서, 읽기 편한 쪽을 골라 싣고 싶은 유혹이 있었습니다. 하지만 출처를 잘못 달면 그게 바로 앞서 고친 문제와 같은 잘못입니다.
- **결과:** 11건만 세부관리기준과 일치했고, **인건비 협약종료 서류·지급수수료 세부(멘토링비, 시험·인증비 등) 10건은 통합관리지침에만** 있었습니다. 기존 인용이 옳았고 원문만 빠져 있던 것이 확인됐습니다.

### 7. OCR이 필요한지 먼저 확인
- **무엇을:** 규정DB에 쓰이는 PDF 5개가 스캔본인지 아닌지 글자 수로 진단했습니다.
- **왜:** 표가 깨져 보이길래 "글자를 못 읽는 건가?" 싶어 OCR(이미지 속 글자를 읽어내는 기술) 도입을 검토했습니다.
- **결과:** 5개 모두 쪽당 282~1,116자가 나오는 **텍스트 PDF**였습니다(스캔본 기준은 100자 미만). 글자는 이미 정확히 읽히고 있었고, 문제는 **읽은 글자의 위치 해석**이었습니다. 여기에 OCR을 붙였다면 정확한 글자를 사진으로 되돌렸다가 다시 읽는 셈이라 오히려 나빠졌을 겁니다.

### 9. 팁스 인건비 증빙을 내부·외부로 분리
- **무엇을:** 인건비 세목을 무엇으로 고르든 내부·외부 서류가 함께 나오던 것을, 규정 표대로 갈라 실었습니다.
- **왜:** 외부 인건비를 골랐는데 내부 직원용 `참여연구자 현황표`가 뜨고, 내부를 골랐는데 `외부참여연구자 소속 기관장 확인서`가 딸려왔습니다. **필요 없는 서류를 준비하게 만드는** 안내입니다.
- **원인:** 규정 표는 인건비를 `내부 / 외부 / 연구근접 지원인력` 세 줄로 나눠 적는데, 규정DB는 그 셋을 **한 문구로 묶어** `참여연구자 급여`·`4대보험 기관부담금`·`퇴직급여충당금`·`신규 채용 인력 인건비` 네 항목에 똑같이 달아뒀습니다. 이 넷은 **비용의 종류**라 표에 자기 줄이 없습니다 — 표의 행에 해당하는 건 내부와 외부입니다.
- **결과:**

| 세목 | 요구하는 증빙 |
|---|---|
| 내부 인건비 (기존 / 신규·일반 / 신규·청년 의무채용) | 참여연구자 현황표 · 급여명세서(월별) · 계좌이체증명 · 건강보험자격득실확인서(영리기관 신규채용) · 현금계상 인건비 관리 계획 · 현물부담확인서 **7종** |
| 외부 인건비 | 근로계약서(타 대학 학생연구자는 재학증명서) · 급여명세서 · 계좌이체증명 · 건강보험자격득실확인서 · **소속 기관장 확인서** · 과제참여계약서(프리랜서·개인사업자) **6종** |

- **안 한 것:** 표에 있는 `연구근접 지원인력 인건비`는 세목으로 만들지 않았습니다. 팁스 편성 양식(별지 제1-③호)에 없는 세목이라, 규정에 있다고 화면에 없는 칸을 만들면 편성과 어긋납니다.

## 🔧 수정한 부분

- **수정 전:** PDF에서 표를 뽑으면 문장이 이렇게 깨졌습니다.

  ```
  세금계산서 신용카드 영수증( ),  견적서,  검수조서 증빙사진 포함( ), 거래처
  ```

- **수정 이유:** 괄호가 엉뚱한 자리에 몰려 있어 **무슨 서류를 말하는지 읽을 수 없습니다.** 이 상태로는 규정DB에 실을 수 없습니다.
- **수정 후:** 원문 그대로 복원됩니다. 앱(`src/extract.ts`)과 추출 스크립트가 같은 규칙을 쓰므로, 앱에 문서를 업로드해 추출해도 결과가 같습니다.

  ```
  세금계산서(신용카드 영수증), 견적서, 검수조서(증빙사진 포함), 거래처 사업자등록증, 거래처 통장사본
  ```

- **덤:** 스크립트가 조각을 이어 붙일 때 무조건 띄어쓰기를 넣던 것도 고쳤습니다. `거래처 사 업자등록증`처럼 한 낱말이 갈라지던 것이 사라지고, 문서 전체에서 704줄이 개선됐습니다.

## 😵 어려웠던 점 & 해결

### (1) 테스트는 통과하는데 화면은 그대로
- **문제:** 증빙 원문 토글을 만들고 자동 테스트도 통과했는데, 실제 화면에서는 계속 `원문 미수록`이었습니다.
- **원인:** 테스트는 **내 컴퓨터 안의 규정 파일**로 돌아가고, 실제 화면은 **서버에 올라간 규정**을 씁니다. 같은 이름이면 서버 쪽이 이깁니다. 테스트가 통과한다고 사용자 화면이 같다는 보장이 없었습니다.
- **해결:** 서버를 직접 조회해 차이를 눈으로 확인한 뒤 다시 올렸습니다. 6개 사업 팩을 항목 단위로 대조해 전부 `동일`인 것까지 확인했습니다.

### (2) 원문이 없는 게 아니라, 남의 원문이 붙어 있었다
- **문제:** 예비창업패키지 증빙 근거를 누르면 `제36조(정부출연기관 연구개발비 이관)`라는, 창업지원과 아무 상관없는 조문이 떴습니다.
- **원인:** 근거를 `제36조 <표-10>`처럼 **번호만** 적어뒀는데, 정작 그 번호의 조문은 통합관리지침(다른 문서)의 것이라 예비창업 자료에는 없었습니다. 앱은 자기 자료에서 못 찾으면 상위 규정 자료를 뒤지는데, 하필 거기에 같은 번호의 다른 조문이 있었습니다. 같은 패키지에서 `통합관리지침 제38조제4항` 1건만 멀쩡했는데, **이건 문서 이름이 적혀 있었기 때문**이었습니다.
- **해결:** 빠진 원문을 채워 자기 자료 안에서 해결되게 했습니다. 그리고 "다른 규정을 가리킬 땐 문서 이름을 붙인다"를 가이드와 앱 추출 규칙 양쪽에 남겼습니다. 39건 전부 정상화됐습니다.

### (3) PDF가 괄호를 글자 위에 덧그리고 있었다
- **문제:** PDF에서 뽑은 표의 괄호·쉼표가 전부 줄 끝으로 밀렸습니다.
- **원인:** 글자 조각의 좌표를 하나씩 찍어보니, 쉼표가 **공백과 똑같은 자리(x=220.2)** 에 있었습니다. 한글에서 PDF로 내보낼 때 본문은 부호 없이 한 덩어리로 그리고, 부호는 그 위에 따로 덧그린 것이었습니다. 왼쪽부터 순서대로 이어 붙이면 겹친 부호가 갈 곳을 잃습니다.
- **해결:** 부호의 좌표를 본문 덩어리 안의 **몇 번째 글자인지**로 환산해, 원래 비어 있던 공백 자리를 되찾아 넣었습니다.
- **되돌아온 문제:** 처음엔 판별을 넓게 잡아서 표의 `-` 칸(값이 없다는 뜻)까지 부호로 보고 앞 칸에 붙여버렸습니다. 기존 테스트가 `| 1차년도 | 66,500 | - |` → `1차년도 66,500-` 로 깨지며 잡아줬습니다. **다른 글자 위에 겹친 부호만** 대상으로 좁혀 해결했고, 이 경우를 테스트로 못 박아뒀습니다.

### (4) 마지막 하나만 남기고 앞의 것을 덮어쓰고 있었다
- **문제:** 세목을 골라 근거 원문을 펼치면, 그 세목의 다른 하위 항목 줄이 강조됐습니다.
- **원인:** 근거를 **"조문 번호"로 묶어** 보관하면서 검색 문구를 하나만 들고 있었습니다. 팁스 증빙표는 **조문 하나에 모든 세목이 들어 있어서** 한 세목의 항목 여럿이 같은 번호를 씁니다. 그래서 뒤 항목이 앞 항목의 문구를 덮어쓰고 마지막 하나만 남았습니다. 외부 전문기술 활용비의 항목 순서가 `기술도입비 → 전문가 활용비 → 연구개발서비스 활용비`라, 마지막 것만 살아남은 것입니다.
- **해결:** 항목마다 문구를 모아 두고, 문구별로 제 줄을 찾아 합치도록 바꿨습니다.
- **되돌아온 문제:** 처음엔 문장 조각을 전부 검색에 썼더니 이번엔 **너무 많이** 잡혔습니다. `거래명세서, 구매의뢰서` 같은 어느 비목에나 나오는 서류가 남의 줄을 짚어서, 연구시설·장비비가 연구재료비·연구실 운영비 줄까지 강조했습니다. **조건이 붙은 묶음만**(출장비의 국내/국외처럼) 따로 찾도록 좁혀 해결했습니다.

### (5) 숫자만 맞추면 되는 상황에서 데이터를 고치기로 한 것
- **문제:** 인건비 증빙을 내부·외부로 나눴더니 기존 테스트가 깨졌습니다. 인정 항목이 4개에서 8개로 늘었기 때문입니다.
- **원인:** 화면이 비목을 보여줄 때 **하위 세목의 항목까지 모아서** 보여주는 구조라, 세목에 항목을 넣으면 비목 쪽 개수도 함께 늘어납니다.
- **해결:** 테스트의 숫자를 4에서 8로 고치면 1초면 끝나지만, 그러면 **뭉뚱그린 옛 문구가 그대로 남습니다.** 대신 원래 네 항목의 증빙을 걷어냈습니다. 내용은 이미 내부·외부로 모두 옮겨져 있어 잃는 것이 없고, 결과적으로 인정 항목 8개 중 증빙을 가진 것은 4개(내부 3 + 외부 1)로 정리됐습니다.
