# [스터디] 리액트 상태관리 (2026-07-21)

## 무엇을 공부했나
화면(컴포넌트)끼리 데이터를 주고받는 방법을 공부했습니다.

## 왜 필요했나
로그인한 사용자의 정보를 로그인 화면뿐 아니라 여러 화면에서 함께 써야 했기 때문입니다.

## 쉽게 설명하면
집 안 여러 방을 오가며 쓰는 물건을 각 방에 따로 두면 관리가 번거롭습니다.
대신 '공용 창고'에 한 번만 넣어두고, 필요한 방에서 그때그때 꺼내 쓰는 방식이라고 이해했습니다.

## 실제로 어떻게 적용했나
로그인 정보를 공용 창고에 저장해서, 모든 화면이 그 창고를 참조하도록 만들었습니다.
덕분에 화면마다 로그인 정보를 다시 확인할 필요가 없어졌습니다.

## 더 알아볼 것
- 창고에 넣는 데이터가 너무 많아지면 어떻게 정리하는지
- 창고 대신 더 간단하게 처리해도 되는 경우는 언제인지
