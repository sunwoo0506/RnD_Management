# [스터디] 화면 요소가 겹치거나 잘리는 두 가지 이유 (2026-07-23)

## 무엇을 공부했나

화면에서 **① 요소가 옆 것과 겹치는 경우**와 **② 요소가 잘려서 안 보이는 경우**의 원인이
서로 완전히 다르다는 것, 그리고 각각 어떻게 푸는지를 공부했습니다.

## 왜 필요했나

같은 날 두 가지 증상을 만났는데, 겉보기엔 둘 다 "화면이 이상하다"였지만 원인이 정반대였습니다.

```
증상 ①  연구자 관리 표에서 이름칸과 고용형태칸이 딱 붙어 겹쳐 보임
증상 ②  편성 표 마지막 줄에서 설명 풍선이 아예 안 보임
```

## ① 겹침 — "칸은 내용보다 작아지지 않으려 한다"

표의 칸 너비를 `150px`로 정해도, **그 안에 든 것이 더 크면 칸이 밀려납니다.**
브라우저가 "내용이 잘리는 것보다는 칸을 키우는 게 낫다"고 판단하기 때문입니다.
(격자 레이아웃의 기본 최소 크기가 `auto` — "내용만큼"이라는 뜻입니다.)

여기에 입력칸의 성질이 겹칩니다.

> 글자 입력칸(`<input>`)은 아무 지시가 없으면 **브라우저 기본 너비(약 170px)** 를 고집합니다.
> 날짜 입력칸은 달력 아이콘까지 있어 더 큽니다.

그래서 "칸은 150px, 내용은 170px" → 칸이 밀려나 옆 칸을 침범합니다.

**해결은 두 줄이었습니다.**

```css
/* ① 칸이 내용보다 작아져도 된다고 허락한다 */
.researcher-row > * { min-width: 0; }

/* ② 입력칸은 자기 칸 너비를 따르게 한다 */
.researcher-id input { width: 100%; }
```

여기에 더해 칸마다 **최소 너비**를 정하고(`minmax(150px, 1fr)`), 화면이 그보다 좁아지면
겹치는 대신 **가로 스크롤**로 넘어가게 했습니다.

> 겹쳐서 못 읽는 것보다 스크롤이 낫습니다. "좁으면 줄인다"에는 한계가 있고,
> 그 한계를 넘으면 **스크롤로 넘긴다**가 정답입니다.

## ② 잘림 — "모서리 밖은 자른다"는 설정

카드에는 흔히 이런 설정이 붙습니다.

```css
.budget-editor { overflow: hidden; }   /* 모서리 밖으로 나가는 것은 자른다 */
```

카드의 **둥근 모서리**를 깔끔하게 유지하려고 넣는 설정입니다. 안에 든 표의 배경색이
모서리 밖으로 삐져나오는 것을 막아줍니다.

문제는 이 설정이 **의도한 것까지 자른다**는 점입니다. 계산식을 마우스 올렸을 때 뜨는
설명 풍선은 카드 아래쪽으로 열리는데, 표의 마지막 줄에서는 풍선이 카드 밖으로 나가므로
통째로 잘려 안 보였습니다.

```
┌─ 카드 ─────────────┐
│  … 표 …            │
│  간접비   [상한]    │
└────────────────────┘
     ▼ 풍선이 여기 뜨는데 카드 밖 → 잘림
```

**해결:**

1. 자르기 설정을 풀고(`overflow: visible`), 대신 **마지막 줄의 모서리를 직접 둥글렸습니다.**
   원래 자르기가 해주던 일을 명시적으로 대신한 것입니다.
2. 추가로 **표 아래쪽 두 줄은 풍선이 위로 열리게** 했습니다. 아래로 열면 화면 밖으로
   나갈 수 있기 때문입니다.

```css
.budget-table > *:nth-last-child(-n+2) .cap-pop {
  top: auto; bottom: calc(100% + 6px);   /* 아래가 아니라 위로 */
}
```

## 두 증상을 구분하는 법

| 증상 | 의심할 것 | 확인 방법 |
|---|---|---|
| 요소가 **옆 것과 겹친다** | 칸의 최소 크기, 내용의 기본 너비 | 내용을 짧게 바꿔보면 겹침이 사라지는가? |
| 요소가 **딱 잘려서 안 보인다** | 조상 요소의 `overflow: hidden` | 부모를 거슬러 올라가며 자르기 설정을 찾는다 |

핵심 차이는 이렇습니다.

- **겹침**은 요소가 **자기 자리를 넘어 자란** 것 → 크기를 다스려야 한다
- **잘림**은 요소는 제대로 있는데 **부모가 가린** 것 → 부모를 봐야 한다

증상만 보고 "CSS가 이상하다"고 뭉뚱그리면 엉뚱한 곳을 고치게 됩니다.
