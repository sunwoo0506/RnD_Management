# [스터디] 차트 색은 취향이 아니라 검증 대상이다 (2026-07-23)

## 무엇을 공부했나

차트(파이 그래프)의 색을 고를 때 "예쁜가"가 아니라 **"누구에게나 구분되는가"를
수치로 검사**해야 한다는 것, 그리고 색이 부족할 때 무엇으로 보완하는지를 공부했습니다.

## 왜 필요했나

총괄 대시보드의 편성 구성 파이를 파스텔 톤으로 바꾸는 요청이 있었습니다.
연한 파스텔 8색을 골라 검증 도구에 넣었더니 결과가 이랬습니다:

```
[FAIL] 밝기 범위       — 너무 밝은 색 2개
[FAIL] 채도 하한       — 회색으로 보이는 색 6개
[FAIL] 일반 시각 구분   — 인접한 두 색이 색약이 아니어도 구분 안 됨
```

## 쉽게 설명하면

색에는 세 가지 함정이 있습니다.

1. **색약**: 남성의 약 8%는 빨강·초록 계열 구분이 어렵습니다. 내 눈에 다른 두 색이
   그들에겐 같은 색입니다. 파이 조각이 맞붙어 있으면 경계가 사라집니다.
2. **대비**: 연한 색은 흰 배경에 묻힙니다.
3. **채도**: 채도를 너무 빼면 모든 색이 회색 계열로 수렴합니다.

> 옷 고르기에 비유하면 — 거울로 보면 예쁜 조합이, 흑백 사진으로 찍으면
> 전부 같은 회색일 수 있습니다. 차트는 "흑백 사진으로도 구분되는가"까지
> 따져야 하는 옷입니다.

중요한 건 이걸 **눈으로 판단하지 않고 도구로 계산**한다는 점입니다. 색 사이의
지각적 거리(사람 눈이 느끼는 차이)를 색약 시뮬레이션까지 돌려 수치로 재는
검증 스크립트가 있고, 통과/실패가 명확히 나옵니다.

## 실제로 어떻게 적용했나

- 완전히 연한 파스텔 대신 **채도를 조금 남긴 파스텔**로 조정해 검증을 통과시켰습니다.
  (밝기·채도를 허용 범위 안쪽 끝까지만 밀어붙인 타협점)
- 그래도 흰 배경 대비가 약한 색이 남는데, 이건 규칙상 **색이 아닌 수단으로 보완**하면
  됩니다: 조각에 마우스를 올리면 비목명·금액·비중 툴팁, 범례에 비중 % 병기,
  "표로 보기" 접힘 표.
- 색 배정은 전체 편성 큰 순으로 **고정**했습니다. 필터로 사업을 바꿔도 "인건비 = 파랑"이
  유지돼야 합니다 — 색이 순번이 아니라 대상을 따라가야 혼란이 없습니다.

## 더 알아볼 것

- 다크 모드를 만들 때는 같은 색을 그대로 쓰면 안 되고 어두운 배경용으로 다시
  밟아서(step) 또 한 번 검증해야 한다는 것
- 조각이 9개를 넘으면 색을 늘리지 말고 "기타"로 접는 이유 (색이 늘수록 구분 가능한
  조합이 급격히 줄어든다)
