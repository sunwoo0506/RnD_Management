# [스터디] CSS 특이도 — 나중에 쓴 규칙이 항상 이기는 게 아니다 (2026-07-22)

## 무엇을 공부했나
CSS(화면 꾸미기 규칙)에서 같은 요소에 서로 다른 규칙이 걸릴 때, 무엇이 이기는지 정하는 **특이도(specificity)**를 공부했습니다.
"나중에 쓴 것이 이긴다"는 흔한 오해이고, 실제로는 **얼마나 콕 집어 가리켰는지**가 먼저입니다.

## 왜 필요했나
참여율 입력칸에 `90`을 넣으면 `%` 기호가 숫자 위에 겹쳐 찍혔습니다.
`%`는 입력칸 위에 겹쳐 두고 오른쪽 여백으로 자리를 비워두는 구조인데, 그 여백 지정이 아무리 해도 먹지 않았습니다.
"내가 나중에 썼는데 왜 안 먹지?"에서 막혔고, 원인이 특이도였습니다.

## 쉽게 설명하면
회사에 지시가 두 개 내려왔다고 해봅시다.

- 지시 A: **"3층 직원들"** 은 오후 6시에 퇴근
- 지시 B: **"3층 영업팀 김대리"** 는 오후 8시에 퇴근

김대리는 몇 시에 퇴근할까요? 당연히 **더 콕 집어 말한 B**를 따릅니다.
A가 나중에 붙었더라도 마찬가지입니다. 대상을 더 좁게 지정한 쪽이 이깁니다.

CSS도 똑같습니다. 대상을 가리키는 방법마다 점수가 다릅니다.

| 가리키는 방법 | 예 | 무게 |
|---|---|---|
| 태그 이름 | `input` | 가벼움 |
| 클래스·속성 | `.labor-rates`, `[type='number']` | 중간 |
| id | `#total` | 무거움 |

**중간 무게가 하나라도 더 많으면, 가벼운 것이 아무리 많아도 못 이깁니다.**
(태그 100개를 모아도 클래스 1개를 못 이깁니다 — 자릿수가 다른 셈입니다.)

## 실제로 어떻게 적용했나
문제가 된 두 규칙은 이랬습니다.

```css
.labor-row input[type='number']  { padding: 6px 7px; }        /* 클래스1 + 속성1 + 태그1 */
.labor-rates label input         { padding-right: 19px; }     /* 클래스1 + 태그2 */
```

- 위쪽: 중간 무게 **2개**(클래스 `.labor-row` + 속성 `[type='number']`)
- 아래쪽: 중간 무게 **1개**(클래스 `.labor-rates`)

그래서 제가 나중에 쓴 아래쪽 여백 지정이 통째로 덮였고, 숫자가 오른쪽 끝까지 밀려 `%`와 겹쳤습니다.

해결은 **같은 방식으로 가리키기**였습니다.

```css
.labor-rates label input[type='number'] { padding: 6px 21px 6px 7px; }
```

중간 무게를 2개로 맞추니(클래스 1 + 속성 1) 여백이 살아났고, 겹침이 사라졌습니다.

## 더 알아볼 것
- `!important` — 특이도를 무시하고 강제로 이기게 하는 장치. 편하지만 남발하면 나중에 아무도 못 고치는 코드가 됨. 왜 "최후의 수단"이라 부르는지
- `:not()`, `:is()` 같은 괄호형 선택자가 특이도에 어떻게 계산되는지 (이번 코드에도 `div:not(.settings-save)` 규칙이 의도보다 넓게 걸려 있었음)
- 브라우저 개발자도구에서 어떤 규칙이 이기고 졌는지 눈으로 확인하는 법 (취소선으로 표시됨)
