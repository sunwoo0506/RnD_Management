# [스터디] 자기 자신에 따라 변하는 값 찾기 — 고정점과 이분 탐색 (2026-07-22)

## 무엇을 공부했나
"결과가 다시 입력에 영향을 주는" 계산을 어떻게 푸는지 공부했습니다.
이런 계산은 공식 한 번으로 답이 안 나오고, **답을 찍어보고 확인하며 범위를 좁히는 방법(이분 탐색)** 으로 풉니다.

## 왜 필요했나
간접비 막대바를 화면에 표시된 상한(예 900만 원)까지 끌면, 도착하는 **그 순간 "상한 초과"** 가 떴습니다.
상한을 지키려고 상한까지 끌었는데 초과가 되니 사용자 입장에서는 버그로만 보입니다.

## 쉽게 설명하면
규정을 그대로 옮기면 이렇습니다.

```
간접비 상한 = 직접비 × 10%
직접비     = 총사업비 − 간접비 − 위탁연구개발비
```

두 줄을 겹쳐 보면 이상한 점이 보입니다.
**간접비를 정하려면 직접비를 알아야 하는데, 직접비를 알려면 간접비를 알아야 합니다.**

> 뷔페에서 "남은 음식의 10%까지 더 담을 수 있다"는 규칙과 같습니다.
> 내가 담을수록 남은 음식이 줄고, 그러면 담을 수 있는 양도 같이 줄어듭니다.
> "지금 남은 것의 10%"를 그대로 담으면, 담고 난 뒤엔 이미 한도를 넘어 있습니다.

이렇게 **자기 자신을 참조하는 값**에서, 더 이상 움직이지 않고 딱 맞아떨어지는 지점을 **고정점**이라고 합니다.
`내가 담은 양 = 담고 난 뒤 기준으로 계산한 한도` 가 되는 바로 그 지점입니다.

### 어떻게 찾나 — 스무고개처럼
1~1000 사이 숫자를 맞히는 스무고개를 떠올려보세요. 하나씩 세지 않고 이렇게 합니다.

1. 가운데인 500을 찍는다 → "더 작아요"
2. 그럼 1~500 사이. 가운데 250을 찍는다 → "더 커요"
3. 250~500 사이. 가운데 375를 찍는다 → …

한 번 물을 때마다 **범위가 절반**이 되므로, 1000개 중에서도 10번이면 답에 닿습니다.
이것이 **이분 탐색**입니다.

상한 문제도 똑같이 풀 수 있습니다. 값을 하나 찍고 "이 금액을 넣었을 때 여전히 상한 안인가?"를 확인합니다.

- 안이면 → 더 넣을 수 있으니 아래쪽 절반은 버리고 위로
- 넘으면 → 너무 많으니 위쪽 절반을 버리고 아래로

이걸 반복하면 **넘지 않는 가장 큰 금액**에 정확히 도달합니다.

## 실제로 어떻게 적용했나
총사업비 1억 원, 위탁·국제공동 각 1천만 원인 과제에서:

- 화면에 보이던 상한: **9,000,000원** → 여기까지 끌면 기준이 줄어 곧바로 초과
- 손으로 풀면: `v = (8,000만 − v) × 10%` → `v = 7,272,727원`
- 이분 탐색으로 찾은 값: **7,272,727원** ✅ 정확히 일치

이제 막대바를 끝까지 끌어도 이 금액에서 멈추고 "상한 초과"가 뜨지 않습니다.
막대바 아래에 `이 비목은 상한 7,272,727원까지`라고 미리 알려주기도 합니다.

**공식을 직접 풀지 않고 이분 탐색을 쓴 이유**는, 규정마다 빼는 항목이 다르기 때문입니다.
(어떤 규정은 현물을 빼고, 어떤 규정은 국제공동·부담비까지 뺍니다.)
규정별로 방정식을 일일이 세우는 대신, **"넣어보고 확인한다"** 는 한 가지 방법으로 전부 처리할 수 있습니다.
계산은 약 27번 반복되지만 사람이 느낄 시간은 아닙니다.

- 이분 탐색이 통하려면 "값이 커질수록 한도는 줄거나 그대로"라는 **단조성**이 필요하다. 이 전제가 깨지는 규정이 나오면 어떻게 할지
