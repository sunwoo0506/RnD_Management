# 총괄 대시보드 — 한눈에 보기 개편 기획 (확정)

여러 과제를 동시에 수행하는 상황에서, 한눈에 보기를 "현재 과제 하나"가 아니라
**등록된 과제 전체를 가로지르는 총괄 대시보드**로 바꾼다. 과제별 상세는 지금처럼
각 화면(예산편성·집행증빙)이 담당하고, 총괄에서 행을 누르면 그 과제로 전환한다.

2026-07-23 사용자와 문답으로 확정. 각 섹션의 ⚑ 표시는 사용자가 직접 고른 결정.

## 화면 골격

```
한눈에 보기 (개편) — R&D 총괄 대시보드
├─ ① 전체 R&D 총괄 현황      지표 카드 + 주관기관 칩 + 과제 목록 표
├─ ② 사업별 사업비 구성       과제 체크박스 표 (재원별 편성/사용)
├─ ③ 사업비 편성 구성 그래프   도넛 + 100% 누적막대 (항상 전체 기준)
└─ ④ 지금 확인할 일          월별 계획 체크리스트 · 증빙 빠짐 · 참여율 현황표
```

기존 한눈에 보기의 콘텐츠 이동: 비목별 집행현황 **제거**(⚑), 증빙 준비 현황 → ④-2로
흡수, 정산 D-day → ① 과제 표로 흡수.

## ① 전체 R&D 총괄 현황

- 지표 카드: 과제 수(진행 중 N / 전체 M) · 전체 사업비 합계 · 지원금 합계 · 미완료 증빙 총 건수
- **주관기관별 과제 수 칩** (중기부 3건 · 산업부 1건 …) — 클릭하면 그 부처 과제만 필터.
  ⚑ 3책 5공 확인이 목적: 부처별 과제 수가 보여야 한다.
- 과제 목록 표: 사업명(programName) / 과제명 + 간략요약 / 주관기관(agency) /
  기간 + 진행률 바 / 총사업비 / 지원금 / 상태
- ⚑ **간략요약 = 새 필드 `Project.summary`** — "이 과제가 무엇을 개발하는지" 직접 입력.
  입력 위치: 과제 설정 + 위저드 1단계(선택). 비어 있으면 "요약 미입력" 힌트.
- ⚑ 기관명 = 주관부처·전문기관(`agency`). 수행 기업명이 아니다.
- ⚑ 종료 과제: 합계에 **포함**하되 "종료" 배지를 달고 목록 뒤로 배치.
- 행 클릭 → 그 과제로 전환 후 관련 화면 이동.

## ② 사업별 사업비 구성

집행증빙의 예산집행현황표에서 월 열을 체크박스로 켜고 끄듯, 과제를 체크박스로
켜고 끄면 합계 행이 따라 바뀌는 표.

- 열: 총사업비 · 지원금(편성/사용) · 민간 현금(편성/사용) · 민간 현물(편성/사용) ·
  미구분 사용액 · 집행률. 편성은 `fundingBreakdown`, 사용은 재원별 집행 합계.
- ⚑ **재원 사용액은 집행마다 직접 입력** — `Expense.fundingSource` 신설
  (`'subsidy' | 'matching_cash' | 'matching_inkind'`).
  - 집행 등록 폼 ③ 집행 내용에 라디오 3개(지원금/민간 현금/민간 현물), 기본값 지원금.
  - 기존 집행건은 재원이 비어 "미구분"으로 집계. 집행건 수정에서 소급 입력하고,
    ② 표의 미구분 열이 0이 될 때까지 보이게 해 소급을 유도한다.
- 재원별로 편성 대비 사용이 초과하면 경고 (민간 현금을 편성보다 많이 쓰면 정산 문제).

## ③ 사업비 편성 구성 그래프

- 기존 "비목별 집행현황" 섹션은 제거하고 **편성 구성** 그래프로 대체(⚑).
- 전체 편성 도넛(비목 기준) + 사업별 100% 누적 가로막대(비목 기준).
  ⚑ 막대(사업 하나)를 클릭하면 그 사업의 세목 구성으로 드릴다운.
- 비목이 기본 축인 이유: 비목 코드는 사업 간 통일돼 정확히 합쳐지지만,
  세목 이름은 자유 입력이라 사업 간 비교가 부정확하다. 세목 비교는 하지 않는다.
- ⚑ 그래프는 ② 체크박스와 **연동하지 않고 항상 전체 과제 기준 고정**.
  ⚑ 화면에 **"전체 사업 기준" 라벨을 명시**해 필터가 안 먹는다는 걸 화면이 말하게 한다.
- 구현 시 dataviz 스킬을 먼저 읽는다.

## ④ 지금 확인할 일

### ④-1. 월별 계획 체크리스트

- ⚑ **금액 자동 판정**: 그 달 집행액 ≥ 계획액 → "진행됨", 미달 → "할 것".
  별도 저장 필드 없음 — 항상 실제 데이터와 일치한다.
- 이번 달 + 지난달까지의 미달분을 함께 나열해 밀린 것이 사라지지 않게 한다.
- [다음달로 미루기]: 남은 계획액(계획−집행)을 다음 달 계획에 더하고 이번 달 계획을
  집행액으로 줄인다 — 기존 `setMonthlyPlan` 재사용, 데이터 구조 변경 없음.
- [집행 등록 바로가기]: 그 과제 집행증빙 화면으로 전환 + 해당 비목·세목 선택.

### ④-2. 증빙 빠짐

- `evidenceReadiness`를 과제마다 돌려 합산. 사업별 ▸ 세목별 그룹 + 바로가기.

### ④-3. 연구인력 참여율 현황표

- 행 = 사람, 열 = 과제별 참여율 + 외부 참여율 + 총 참여율 + 과제 수(책임/전체).
- 사람 합치기는 **이름 기준** (공통 인물 ID 없음). 표기가 다르면 다른 사람으로
  집계된다는 안내를 화면에 단다.
- 총 참여율 = 등록 과제 projectRate 합 + 외부 참여율. 100% 초과 시 경고.
  외부 참여율이 과제마다 다르게 입력됐으면 **최대값 하나만** 쓴다 (중복 합산 방지).
- ⚑ **`Participant.isLead`(연구책임자) 신설** — 인력·담당자 화면에서 과제별 지정.
  현황표가 인당 "책임 N건 / 전체 M건"을 세고, 3책(책임 3건 초과)·5공(전체 5건 초과)
  경고를 낸다.

## 데이터 변경 요약

| 필드 | 위치 | 용도 |
|---|---|---|
| `summary?: string` | Project | ① 간략요약 (무엇을 개발하는지) |
| `fundingSource?: 'subsidy' \| 'matching_cash' \| 'matching_inkind'` | Expense | ② 재원별 사용액 |
| `isLead?: boolean` | Participant | ④-3 3책 5공 |

모두 선택 필드라 기존 저장 데이터와 호환된다. storage.ts의 필드 검증에 추가 여부 확인.

## 구현 순서 (제안)

1. **①** 총괄 현황 + `summary` 필드 — 화면 골격을 세우는 단계, 다른 섹션과 독립
2. **③** 편성 그래프 — 읽기 전용, 기존 데이터만으로 가능
3. **②** 재원 구성 — `fundingSource` 폼 변경 포함
4. **④-2** 증빙 빠짐(쉬움) → **④-1** 월별 체크리스트 → **④-3** 참여율(`isLead` 추가)

단계마다 테스트·빌드를 통과시키고 따로 커밋한다.
