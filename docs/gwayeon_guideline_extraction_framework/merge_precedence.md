# 규정 병합과 우선순위

## 1. 두 가지 순위를 분리한다

### 법적 효력 순위

```text
법률
> 시행령
> 시행규칙
> 고시·훈령·예규
> 지침·매뉴얼
> FAQ·질의회신
> 기관 자체규정
```

### 적용 구체성 순위

```text
협약별
> 사업별 공고·별첨
> 전문기관별
> 부처별
> 국가 공통
```

법적 효력이 낮은 문서가 상위 법령에 위반되는 규칙을 만들 수는 없습니다.
하지만 상위 규정이 허용한 범위 안에서는 더 구체적인 사업 규칙을 우선 적용할 수 있습니다.

---

## 2. 권장 우선순위 점수

| 범위 | priority |
|---|---:|
| 협약별 | 1000 |
| 사업 공고별 | 900 |
| 전문기관별 | 700 |
| 부처별 | 500 |
| 국가 공통 지침 | 300 |
| 기관 자체규정 | 200 |
| FAQ·참고자료 | 100 |

실제 병합은 priority만 보지 않고 다음을 모두 확인합니다.

1. 시행일
2. 적용 기관
3. 적용 사업
4. 과제 속성
5. 법적 효력
6. 예외 조건
7. 사용자 협약일 또는 집행일

---

## 3. 병합 결과

### REPLACE

사업 공고가 공통 상한을 명확히 다른 값으로 정한 경우

### NARROW

공통 허용 범위보다 더 좁은 범위를 적용하는 경우

### EXTEND

상위 규정이 허용한 범위 안에서 사용 항목을 추가하는 경우

### ADD_CONDITION

금액은 같지만 추가 승인 또는 증빙을 요구하는 경우

### DISABLE

특정 사업에서 비목 사용을 금지하는 경우

---

## 4. 충돌 처리

다음 경우 자동 결정하지 않습니다.

- 공고가 상위 지침보다 넓은 집행을 허용
- 협약과 공고의 상한이 다름
- 부칙의 경과조치가 불명확
- FAQ가 고시 문언과 다름
- 기관 자체규정이 국가사업에만 더 높은 단가를 적용

이 경우 `CONFLICT_REVIEW_REQUIRED`로 반환합니다.
