# 비목·항목별 증빙 데이터 현황 (2026-07-22)

집행 화면의 증빙 안내를 규정DB 기반으로 바꾸기 위한 조사와 그 결과.
**작업 완료 — 아래 1~5단계를 모두 반영했다.** (남은 것은 §5의 후속 과제)

## 0. 구조

| 데이터 | 출처 | 화면 |
|---|---|---|
| `category.requiredDocs` | `convert-regulation-db.mjs`의 `DOCS_BY_CODE` **하드코딩** | 집행 화면 "자동 안내 증빙 · 앱 기본 예시" |
| `category.evidenceRules` | 규정DB `regulation_rules.required_documents` | 집행 화면 "규정이 요구하는 증빙" + 기준 패널 |
| `allowedItems[].evidence` | 규정DB `expense_allowed_items.evidence_summary` | 집행 화면 "세부 항목별 증빙" + 인정 항목 목록 |

- 규정 증빙은 체크리스트에 **자동으로 넣지 않는다.** "10만원 초과/이하 회의비"처럼
  조건이 갈리는 규칙이 있어, 함께 넣으면 서로 어긋나는 서류를 동시에 요구하게 된다.
  조건과 근거를 보여주고 해당하는 것만 눌러 담는다.
- 공고가 정하지 않은 증빙은 `basePackId`의 상위 규정에서 물려받되 출처를 나눠 표시한다.

## 2. 사업별 상세

### 팁스 (TIPS) — 가장 완비된 원문

- **원문**: `붙임1. 2026년 팁스TIPS 총괄 운영지침 2차 개정안 본문.hwp`
- **위치**: `11.다.1) 비목별 증빙서류` (80쪽) — 추출 텍스트 3,301~3,630행
- **형태**: `세목 | 세세목 | 증빙서류` 3단 표, 약 18~20행
- **덮는 범위**: 인건비(내부/외부/연구근접지원인력), 학생인건비, 연구시설·장비비,
  연구재료비, 연구활동비 9개 세목(지식재산창출·외부전문기술·회의비·출장비·
  소프트웨어활용비·연구실운영비·연구인력지원비·해외연구자유치·종합사업관리비·그 외),
  연구수당, 위탁연구개발비, 국제공동연구개발비, 간접비
- **매핑**: 세세목이 앱의 `allowedItems`와 이름이 거의 같아 1:1로 붙는다
- 바로 뒤 `11.다.2) 연구개발비 불인정 기준` 표(3,631~3,885행)는 아직 미반영 (§4 참조)

### 예비창업패키지 — 표 있음

- **원문**: `(별첨4) 중소기업창업 지원사업 통합관리지침 제14차.pdf` (55쪽)
- **위치**: **제36조** (16~17쪽)
- **형태**: `비목 | 비목 정의 | 증빙서류` 3단 표
- **특징**: 지급수수료가 세부 항목별로 쪼개져 있다 —
  기술이전비 / 학회·세미나 참가비 / 전시회·박람회 참가비 / 시험·인증비 / 멘토링비 /
  기자재임차비 / 장비수리비 / 사무실임차료 / 운반비 / 보험료 / 보관료 / 회계감사비 /
  해외 액셀러레이터 보육 참가비 / 규제애로 해소 법률컨설팅비
  → 앱의 `allowedItems` 단위와 잘 맞는다
- 인건비·건강검진비·여비(국내/해외)·교육훈련비·광고선전비·기타(비교견적서)도 포함
- 제36조 ②항: 명시되지 않은 증빙은 사업별 세부관리기준을 따르고 추가 요청 가능
- **PDF 텍스트 추출 주의**: 한글 PDF라 어순이 섞여 나온다(`pdfjs-dist`로 추출 가능).
  표 구조는 사람이 확인하며 정리해야 한다

### 국가연구개발비 사용 기준 — 항목별 표 없음

- **원문**: `국가연구개발사업 연구개발비 사용 기준(고시)(제2026-38호).hwpx`
- 증빙 언급 16곳이지만 대부분 보관·정산 절차. **비목별 조항은 8곳**
- 이미 추출됨(5건): 회의비 2, 국외출장 2, 위탁연구개발비 계좌이체증명 1
- **작업으로 반영한 나머지**
  - 연구시설·장비/연구재료 구매 시 **검수확인서** (100만원 이하 검수 간소화 조항 포함)
  - **연구개발부담비** 계좌이체증명 (제42조 ②)
  - **국제공동연구개발비** 계약서 + 계좌이체증명 (인정항목엔 있으나 규칙으로는 없음)
  - **연구혁신비**는 연구비통합관리시스템 입력으로 증명자료 갈음
- 이 고시는 원래 항목별 증빙을 정하는 문서가 아니다 → 여기서 더 나올 게 없다

### 디딤돌 — 원문이 없다

- 관리지침 첫 줄: *"중소기업기술개발 지원사업 관리지침(공통지침)에 따라 운영함을
  원칙으로 하되, 세부과제별 특성에 따라 별도로 정함이 있는 경우 그 내용을 우선"*
- 관리지침 자체에는 증빙 표가 없다 (증빙 언급 2곳, 모두 신청 자격 관련)
- **공통지침 원문을 구해야 한다.** 그때까지는 상위 규정(국가연구개발비)의 증빙을 물려받아 쓴다

## 3. 진행 결과

1. ✅ **화면** — 집행 화면에 규정 증빙 블록·세부 항목별 증빙 추가, 인정 항목에 증빙 표시,
   앱 예시 기본값에 "앱 기본 예시" 표시
2. ✅ **팁스** — 지침 11.다.1) 표를 인정 항목 35/36건에 반영
   (보유 장비 사용료 현물은 표에 해당 행이 없어 비워둠)
3. ✅ **예창패** — 제36조 표를 인정 항목 21/21건에 반영 + "2천만원 이상 거래 시 비교견적서"를 규칙으로
4. ✅ **국가연구개발비** — 제22조2·3항, 제28조4항, 제42조2항, 제25조의2제5항 반영
5. ✅ **디딤돌** — 공통지침 원문은 없지만 상위 규정(국가연구개발비)의 증빙을 물려받게 함

### 작업 중 찾은 변환기 버그 두 가지

- `category_code: 'ALL'`인 증빙 규칙이 **어느 비목에도 붙지 못하고 사라졌다.**
  증빙 규칙으로 분류돼 금지·주의 목록에서도 빠지기 때문. 팁스 2건이 유실 중이었다.
- 증빙 규칙에 **기관 유형(institution_scope) 필터가 없어** 비영리 전용 증빙이
  영리기관 팩에도 실렸다.

## 4. 남은 후속 과제

- **팁스 `11.다.2) 연구개발비 불인정 기준` 표** — 일반기준 13개 + 비목별 목록.
  비목 단위 긴 목록이라 지금 구조(`restriction_summary`는 항목 단위, `warning` 규칙은
  건당 한 줄)에 그대로 넣으면 화면이 무너진다. 비목별 "불인정 기준" 블록을 따로 만드는
  설계가 먼저 필요하다.
- **디딤돌 공통지침 원문 확보** — 「중소기업기술개발 지원사업 관리지침」을 별도 패키지로
  만들고 `basePackId` 계층을 한 단계 더 태운다 (디딤돌 → 공통지침 → 국가연구개발비).
- **국가연구개발비 인정 항목 142개**는 여전히 항목별 증빙이 없다. 고시에 없는 내용이라
  다른 문서(전문기관 정산 매뉴얼 등)를 붙이지 않는 한 채울 수 없다.

## 5. 참고

- 추출 텍스트 재생성: `node scripts/extract-hwp-text.mjs <파일>` (hwp/hwpx)
- PDF: `pdfjs-dist`의 `getDocument`로 페이지별 텍스트 추출
- 규정DB → 팩 변환: `node scripts/convert-regulation-db.mjs docs/extraction_DB/<폴더>`
- 서버 반영: `node scripts/upload-regulation-db.mjs docs/extraction_DB/<폴더>`
