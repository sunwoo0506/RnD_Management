# 내 프로젝트 진단 결과

**진단일:** 2026-07-16 | **프로젝트:** 과제온 (project03)

---

## 1. 현재 프로젝트 한 줄 요약

정부 R&D(중기부) 과제를 처음 수행하는 1~5인 스타트업 담당자가 **따라만 하면 예산 편성·집행·증빙·변경에서 정산 실수를 예방**할 수 있게 돕는 가이드형 웹앱.

## 2. 현재 상태 분류

- **유형: 2. 방향 명확 + 오버스펙**
- **이유:**
  - **방향은 명확하다.** PRD에 타깃(1~5인 스타트업의 겸직 담당자), 문제(정산 불인정 = 기업 자체 부담), 근거 데이터(커뮤니티 채팅 1,490건 분석), 검증 가설("따라만 하면 실수가 줄어든다")까지 구체적으로 정리되어 있다. 비개발자 MVP 중 기획 완성도가 높은 편이다.
  - **기술은 오버스펙이다.**
    - 같은 앱이 **React(src/)와 Streamlit(app.py) 두 벌**로 존재한다. 현재 app.py는 React 빌드 결과를 iframe으로 띄우는 껍데기 역할이고, 그 아래 Streamlit 구현은 실행되지 않는 죽은 코드다.
    - 코드를 거의 이해하지 못하는 상태(AI 전적 의존)에서 **로그인 + 서버 DB**까지 이번 수업 안에 넣으려 한다. 남은 2~3주와 충돌한다.
    - 배포도 GitHub도 없어서, 정작 검증하려는 가설("따라 하면 실수가 준다")을 **실제 사용자에게 확인할 수단이 아직 없다.**
    - 이미 있는 `src/storage.ts`(localStorage 저장)로 서버 DB 없이도 "새로고침하면 데이터가 사라지는" 문제를 해결할 수 있다. DB가 필요하다고 느낀 문제의 상당 부분은 이미 풀려 있다.

## 3. 유지할 것

- **핵심 흐름:** 과제 등록 → 예산 초안 자동 생성 → 비율 초과 경고 → 집행 등록 → 증빙 체크리스트 자동 표시. 이 한 줄기가 서비스의 존재 이유이고, 이미 동작한다.
- **React 앱(src/)** — 본인이 이어가기로 선택한 스택. 정적 배포가 가능해서 서버 없이도 공개할 수 있다.
- **localStorage 저장(src/storage.ts)** — 서버 DB의 90% 역할을 공짜로 해준다.
- **PRD의 문제 정의와 규정 데이터(비목별 상한·증빙 목록)** — 이 도구의 진짜 자산은 코드가 아니라 이 데이터다.

## 4. 줄일 것

- **"P0 3개 전부" → "핵심 흐름 1개를 매끄럽게 + 변경 비교표"**로 축소. 완성 경험이 목표라면 기능 수가 아니라 한 줄기의 완성도가 기준이어야 한다.
- **공문 Word 자동 생성(P0-3)** → 변경 전/후 비교표까지만 자동으로 만들고, 공문은 빈칸 채우기 템플릿 다운로드로 단순화.
- **증빙 파일 업로드** → 파일 저장 대신 "체크박스로 완료 표시"만으로도 가설 검증에는 충분하다.

## 5. 버릴 것 (이번 수업 범위에서)

- **로그인 / 계정당 2명** — 검증 단계에서는 링크 공유로 충분하다.
- **서버 DB** — localStorage로 대체. DB·로그인은 베타 사용자가 "계속 쓰겠다"고 한 뒤(2단계)에.
- **이메일 알림 발송(D-30/14/7)** — 발송 인프라 없이 화면 내 D-day 표시로 대체.
- **Streamlit 중복 구현(app.py)** — React로 가기로 했으니 삭제. 두 벌 유지가 가장 큰 낭비다.
- **잡파일 정리** — `.chrome-verify/`, `*.tsbuildinfo`, 스크린샷 png 등은 저장소에서 제외(.gitignore).

## 6. 다시 정해야 할 것

- **"완성"의 정의:** 기능 개수가 아니라 **"배포된 링크를 남에게 보내서 혼자 써보게 할 수 있는 상태"**로 재정의할 것.
- **DB·로그인의 시점:** "이번 수업"이 아니라 "베타 테스터 5명이 계속 쓰겠다고 답한 후"로 미룰 것.
- **성공 기준:** PRD의 T-01~T-05(예산 편성~집행 경고)를 배포된 링크에서 통과하면 이번 수업은 성공.

- 배포 링크: 없음 (미배포)

## 2. 지금 되는 것
- 기능 1: 총 사업비 입력 시 중기부 비목별 배분 초안 자동 생성 + 상한 비율 초과 시 경고

- 기능 2: 이메일 알림(D-30/14/7), 로그인/계정, 서버 저장 — 이번 수업 범위에서 제외하기로 진단
